# Demo Flow Documentation

## Overview
This document outlines the demo walkthrough for the Attendance & Late-Stay Copilot system.

## Demo Scenarios

### Scenario 1: Basic Attendance Flow
1. Employee enters office
2. Face recognition system captures entry
3. Attendance auto-recorded
4. Copilot displays attendance summary
5. Employee can view their attendance status

### Scenario 2: Late-Stay Detection
1. Employee remains in office after 8:00 PM
2. System detects late stay (especially for women employees)
3. System logs the event
4. Security team notified (if applicable)
5. Safe exit logged automatically when employee leaves

### Scenario 3: Attendance Summary Request
1. User asks Copilot: "Show my attendance for this month"
2. Copilot retrieves attendance data
3. Summary displayed with:
   - Total days worked
   - Late arrivals
   - Work hours
   - Late stays

### Scenario 4: Work Balance Report
1. Project Manager requests work balance report
2. System analyzes:
   - Project-wise attendance
   - Late-night work patterns
   - Workload distribution
3. Report generated with insights
4. Recommendations provided

### Scenario 5: WFO Compliance Check
1. HR requests WFO compliance report
2. System analyzes attendance data against WFO mandate
3. Compliance report generated
4. Non-compliance cases flagged

### Scenario 6: Shift Allowance Request
1. Employee requests shift allowance
2. System checks:
   - Project eligibility
   - Shift timings
   - Previous requests
3. Request processed or recommendations provided

### Scenario 7: Well-being Recommendations
1. System analyzes employee work patterns
2. Detects frequent late stays or long hours
3. Generates well-being recommendations
4. Displays recommendations to employee
